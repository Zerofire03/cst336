Microsoft System Center Configuration Manager

Main Page Operating System Deployment Operating System Updates Application Deployment


Microsoft System Center Configuration Manager is a very powerful and robust tool that enterprises can use to manage their workstations. It covers all of the main aspects that an organization would want in this kind of software; Operating System Deployment, Operating System updates, and Application Deployment and updating. To be able to fully master SCCM, it can take years of working with it and dedication to fully master all of the intricicies that it has.

Brief History

SCCM can trace it's roots back to Systems Management Server that was first published back in 1994. This version was to be used on MS-DOS Windows NT, and MacOS. The 2007 version of the product was the first year to use the name System Center Configuration Manager. Up until the year 2015, there was a new major version released about the time of each major Windows kernel update. Starting in 2015 the software is no longer released with a year anymore. It is now considered a Current Branch with a build number. The developers have moved to a much more rapid release cycle with multiple major versions of the software coming out each year; as many as 4 in one year.

Future

The future of the product is looking good. Microsoft has commited to supporting the softwre for quite some time. There is a dedicated team of developers that are working on the software, with updates/bug fixes and new features coming out very rapidly to support the very agile market that this software falls into.
